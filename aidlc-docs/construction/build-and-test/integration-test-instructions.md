# 結合テスト・手動テスト手順

## 概要

保活手帳v1は単一ユニットのクライアントサイドアプリのため、結合テストは手動でのユーザーフローテストとして実施する。

## 前提

```bash
npm run dev
```

ブラウザで http://localhost:3000 にアクセスした状態で実施。

## テストシナリオ

### シナリオ1: 初回起動〜園追加フロー（US-01, US-02, US-03）

1. ブラウザのlocalStorageをクリア（DevTools > Application > Storage > Clear site data）
2. http://localhost:3000 にアクセス
3. **確認**: オンボーディングダイアログが表示される
4. **確認**: 「できること」リストと「データの保存について」が表示される
5. 「はじめる」をタップ
6. **確認**: 園一覧画面が表示される（空状態メッセージあり）
7. **確認**: ページをリロードしてもオンボーディングが再表示されない
8. 「園を追加する」ボタンをタップ
9. 園名に「テスト園A」、見学日に本日の日付を入力
10. 「追加」をタップ
11. **確認**: 園一覧に「テスト園A」が表示される

### シナリオ2: 園詳細の編集フロー（US-04, US-06, US-07, US-08）

1. 園一覧から「テスト園A」をタップ
2. **確認**: 園詳細画面が表示される
3. 園名の「編集」をタップ → 「テスト園B」に変更 → 「保存」
4. **確認**: 園名が「テスト園B」に変わる
5. 見学日の「編集」をタップ → 日付を変更 → 「保存」
6. **確認**: 見学日が更新される
7. メモの「編集」をタップ → テキストを入力 → 「保存」
8. **確認**: メモが表示される
9. ページをリロード
10. **確認**: 変更が全て保持されている

### シナリオ3: 見学のコツ（US-05, US-12）

1. 初回の園詳細画面遷移時
2. **確認**: 見学のコツダイアログが自動表示される
3. ダイアログを閉じる
4. 別の園の詳細に遷移
5. **確認**: コツダイアログは再表示されない
6. 「見学のコツを見る」リンクをタップ
7. **確認**: コツダイアログが再表示される

### シナリオ4: 園の削除（US-09）

1. 園詳細画面で削除アイコン（ゴミ箱）をタップ
2. **確認**: 削除確認ダイアログが表示される
3. 「キャンセル」をタップ
4. **確認**: 園は削除されない
5. 再度削除アイコンをタップ → 「削除」をタップ
6. **確認**: 園一覧に戻り、その園が表示されない

### シナリオ5: ヘルプ再表示（US-13）

1. 園一覧画面のヘルプ[?]アイコンをタップ
2. **確認**: オンボーディングと同じ内容のダイアログが表示される
3. ダイアログを閉じる
4. **確認**: 通常の園一覧に戻る

### シナリオ6: 空文字バリデーション

1. 園追加画面で園名を空にして「追加」をタップ
2. **確認**: 追加ボタンが非活性、または追加されない
3. 園詳細画面で園名を空にして「保存」をタップ
4. **確認**: エラーメッセージ「園名を入力してください」が表示される

### シナリオ7: Cookie同意

1. localStorageをクリア
2. ページをリロード
3. **確認**: Cookie同意バナーが画面下部に表示される
4. 「同意しない」をタップ
5. **確認**: バナーが消える
6. ページをリロード
7. **確認**: バナーが再表示されない

### シナリオ8: PWA インストール

1. Chrome で https 環境（Vercelデプロイ後、またはローカルHTTPS）にアクセス
2. **確認**: アドレスバーにインストールプロンプトが表示される
3. モバイルの場合: 「ホーム画面に追加」が利用可能
4. インストール後、スタンドアロンモードで起動することを確認

## レスポンシブ確認

以下の画面幅で表示を確認:
- **モバイル**: 375px（iPhone SE）
- **モバイル大**: 430px（iPhone 15 Pro Max）
- **タブレット**: 768px（iPad）

全画面で操作に支障がないことを確認する。
