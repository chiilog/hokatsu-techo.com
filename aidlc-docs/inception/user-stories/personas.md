# Personas - 保活手帳 v1 SLC

## Primary Persona: 一般的な保活中の保護者

| 項目 | 内容 |
|------|------|
| **名前** | マイ（代表的ユーザー像） |
| **属性** | 30代、共働きまたは育休中、初めての保活 |
| **デバイス** | iPhone（日常的に使うアプリはLINE、Instagram程度） |
| **見学数** | 3〜5園程度 |
| **見学期間** | 数ヶ月 |
| **ITリテラシー** | SNS閲覧が中心。スプレッドシートは名前を聞いたことがある程度。アプリのインストールや設定は苦手意識がある |

### Goals
- 難しいことを考えずに、見学した園のことをメモしたい
- 「このアプリ何？」とならずに、すぐ使い始めたい
- パートナーに「こういうアプリで記録してる」と見せたい

### Pain Points
- そもそもメモをどうやって残すか手段を知らない（紙かLINEの自分メモ）
- アプリの初期設定や操作説明が長いと離脱する
- 「ホーム画面に追加」の意味がわからない（PWAの概念がない）
- 専門用語や英語が出てくると不安になる
- 見学中は片手しか使えないことが多い（子どもを抱っこ等）
- 複数園を見学すると記憶が混ざり、申込時に困る

### Context of Use
- **見学前**: 自宅で落ち着いて園をリストアップ（両手使える）
- **見学中**: 園内で説明を聞きながら（片手、短時間の操作）
- **見学直後**: 帰り道や自宅で印象を書き留める（両手使える）
- **申込時**: 自宅で複数園のメモを見比べる（両手使える）

### Success Criteria
- 開いた瞬間に何をするアプリかわかる
- 迷わず園を追加してメモを書ける
- 次の見学でも使いたいと思える
- 申込時にメモを見返して記憶が蘇る

### Design Implications（v1設計への示唆）
- ボタンやラベルは平易な日本語で、専門用語を避ける
- 操作に迷わない最小限のUI（選択肢を減らす）
- オンボーディングは短く直感的に
- エラーメッセージは原因と対処を具体的に示す
- PWAのインストール誘導は視覚的にわかりやすく

## Secondary Persona: ITリテラシーの高い保護者

| 項目 | 内容 |
|------|------|
| **名前** | ユイ（代表的ユーザー像） |
| **属性** | 30代、共働き、0〜1歳児の保活中 |
| **デバイス** | iPhone（主にスマホのみで利用） |
| **見学数** | 5〜10園程度 |
| **見学期間** | 数ヶ月（6月〜8月が多い） |
| **ITリテラシー** | 日常的にさまざまなアプリやWebサービスを活用。スプレッドシートで管理を試みた経験あり |

### Goals
- 見学した園の印象を、申込時まで鮮度高く覚えておきたい
- 見学中や見学直後に、サッと記録を残したい
- 後から見返して「そうそう、こういうとこだった」と思い出したい

### Pain Points
- スプレッドシートはスマホで使いにくく破綻する
- 紙のメモは字が汚くて読み返す気にならない
- 既存のメモアプリは保活に特化しておらず、情報が散らばる
- 見学中は片手しか使えないことが多い（子どもを抱っこ等）

### Key Differences from Primary
- スプシや他のツールを試して「使いにくい」と判断した上で保活手帳にたどり着く
- UIが多少シンプルすぎても操作で困ることはない
- PWAの「ホーム画面に追加」は説明なしでも対応できる
- Primary（マイ）に合わせた設計であれば自然とカバーできる
