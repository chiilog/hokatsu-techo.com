# Requirements Document - UX改善（iOS風UI・自動保存）

## Intent Analysis

| 項目 | 内容 |
|------|------|
| **User Request** | iOS風UIへの刷新、自動保存、見学日入力の改善 |
| **Request Type** | Enhancement（既存機能のUX改善） |
| **Scope** | Multiple Components（園詳細・園追加・ナビゲーション・トースト） |
| **Complexity** | Moderate（UI/UXパターンの変更、画面遷移モデルの変更） |
| **設計原則** | 「触る人が慣れているUIを踏襲する」— Primary Persona マイが難なく使える |

---

## 1. Functional Requirements

### FR-UX-01: iOS風の編集フロー（ハイブリッド保存モデル）

現在の「インライン編集 + 保存ボタン」パターンを廃止し、iOS風のナビゲーション型編集に変更する。

- 園詳細画面では、園名・見学日・メモの各フィールドを「chevron right（>）」付きの行として表示する
- 各行をタップすると、該当フィールドの編集画面に遷移する

**テキスト入力（園名・メモ）— iOS連絡先編集パターン:**
- 編集画面の右上に「完了」ボタンを配置し、タップで保存 + 園詳細画面に自動で戻る
- 変更がある状態で「戻る」を押した場合、破棄確認ダイアログを表示する（「変更を破棄しますか？」→ 破棄 / 編集を続ける）
- 変更がない状態で「戻る」を押した場合、そのまま園詳細画面に戻る

**見学日選択 — 即時保存パターン:**
- カレンダーで日付を選択した時点で即保存される（トグル的操作）
- 「戻る」で園詳細画面に戻る

### FR-UX-02: 控えめな保存フィードバック

- 保存が実行されるたびに、iOS風の控えめなトースト通知を画面下部に表示する
- トーストは1.5〜2秒で自動消去する
- チェックマーク（✓）アイコン付きで視覚的確信を強める
- デザインはさりげなく、操作を邪魔しないスタイルとする
- `role="status"` / `aria-live="polite"` を設定する

### FR-UX-03: ナビゲーションスタイルの変更

- 画面遷移はCSSトランジションによるiOS風アニメーション（左からスライドイン/右へスライドアウト）
- ブラウザのスワイプバックとの競合を避けるため、カスタムスワイプジェスチャーは実装しない
- 戻るボタンはiOS風の「< 戻る」スタイルで画面左上に配置
- 園一覧 → 園詳細 → 各フィールド編集の3階層ナビゲーション
- 編集完了（「完了」タップ / 見学日選択）後は園詳細画面に自動で戻る

### FR-UX-04: 園追加時の見学日入力の改善

- 園追加画面の見学日選択を以下の2択に変更する:
  - 「今日」（タップで本日の日付がセットされる）
  - 「あとで設定する」（`null` = 未定としてセットされる）
- 現在のHTML5日付ピッカーは園追加画面では使用しない

### FR-UX-05: 園詳細での見学日編集

- 園詳細画面の見学日フィールドをタップすると、見学日編集画面に遷移する
- 見学日編集画面では、iOSカレンダーアプリ風のカレンダーピッカーを表示する
- 初期表示: 現在の見学日がある場合はその月、なければ今月
- 過去日付も選択可能（見学済みの日付を記録するため）
- 日付の表示形式: 「2026年4月2日（木）」
- 「未定にする」オプションも提供する（日付をクリアして `null` に戻す）
- 日付を選択した時点で即保存される

### FR-UX-06: フォーム入力スタイル

- 入力フィールドはiOSネイティブ風のスタイルにする
- 園名編集: テキスト入力フィールド1つのシンプルな画面（遷移時に自動フォーカス）
- メモ編集: 複数行テキストエリアのシンプルな画面（遷移時に自動フォーカス）

## 2. Non-Functional Requirements

### NFR-UX-01: ペルソナ適合性

- Primary Persona「マイ」（ITリテラシー低、iPhone中心）が迷わず操作できること
- iPhoneユーザーが普段使っているUI慣習に沿うこと
- 専門用語や英語表記を避け、平易な日本語ラベルを使用すること

### NFR-UX-02: アニメーション・パフォーマンス

- 画面遷移アニメーションは60fps以上を維持すること
- CSSトランジションで実装（View Transitions APIへの移行は将来検討）
- トースト表示・非表示はスムーズなフェードイン/アウトとすること
- アニメーションは `prefers-reduced-motion` メディアクエリを尊重すること

### NFR-UX-03: セキュリティ

- SECURITY拡張ルール（SECURITY-01〜15）を継続適用
- 入力バリデーション（園名の空チェック等）は現行通り維持

### NFR-UX-04: アクセシビリティ

- 画面遷移時のフォーカス管理（編集画面に遷移したらinputに自動フォーカス）
- トーストに `role="status"` / `aria-live="polite"` を設定
- chevronアイコンは装飾のため `aria-hidden="true"` を付与
- 各編集画面に適切な見出し構造（`<h1>` 等）を設定

### NFR-UX-05: エラーハンドリング

- 保存失敗時（localStorage容量超過等）はエラートーストを表示し、編集画面に留まる
- ユーザーの入力データが失われないよう、保存処理はtry-catchで囲む
- エラーメッセージは「保存できませんでした。もう一度お試しください」等の平易な日本語

## 3. スコープ外（今回見送り）

| 項目 | 理由 |
|------|------|
| メモの分離（聞きたいメモ / 当日メモ） | v2以降で検討 |
| 既存データの移行 | 本番未稼働のため不要 |
| リスト表示スタイルの変更（iOS設定アプリ風グルーピング） | 今回はナビゲーションとフォーム入力に集中 |
| View Transitions API への移行 | CSSトランジションで開始、必要に応じて将来切り替え |
