## Intent Analysis

| 項目 | 内容 |
|------|------|
| **User Request** | プロトタイプからのリプレイス。HCDベースの仕様書に基づくv1 SLC構築 |

## 1. Functional Requirements

### FR-01: 園の追加
- 園名（必須）と見学日（任意、デフォルト「未定」）を入力して園を追加できる
- 園名が空の場合は追加ボタンを非活性にする
- メモは追加画面では入力しない（詳細画面で入力）
- 追加後は園一覧に遷移する

### FR-02: 園一覧の表示
- 登録した園をカード形式で一覧表示する
- 並び順は追加順（新しいものが上）固定
- カードには園名、見学日、メモの冒頭プレビューを表示
- メモがない場合はプレビューなし
- 見学日が未設定の場合は「未定」と表示
- 園が0件の場合は空状態メッセージを表示
- カードタップで園詳細画面に遷移

### FR-03: 園詳細の表示・編集
- 園名、見学日、メモを表示する
- 各項目は[編集]タップでインライン編集可能
- 園名は空にできない（バリデーション）
- 見学日は日付ピッカーまたは「未定」を選択
- メモは複数行対応、空でもOK
- メモが空の場合はプレースホルダー「気づいたことを自由に書けます」を表示

### FR-04: 園の削除
- 園詳細画面から削除可能
- 削除ボタンは目立たない位置に配置
- 確認ダイアログ「この園を削除しますか？」→ 削除/キャンセル

### FR-05: 初回オンボーディング
- 初回起動時にアプリの説明を表示
  - このアプリでできること（園を登録してメモを残す）
  - データの保存先の説明（v2以降の変更に対応しやすい文言にする）
- 「はじめる」ボタンで園一覧に遷移
- 次回起動時には表示しない

### FR-06: 見学のコツ（コンテキスト内ガイド）
- 初めて園詳細画面に遷移したときに見学のコツ（3つ）をヒント表示
  1. 先生の様子を見る
  2. 違和感を大事にする
  3. 気づいたことは何でもメモ
- アプリ全体で初回の園詳細遷移時のみ自動表示（2回目以降は表示しない）
- 園詳細画面のメモ欄近くに「見学のコツ」リンクを常時配置し、いつでも再表示可能

### FR-07: ヘルプ
- 園一覧画面のヘッダーにヘルプアイコン[?]を配置
- タップでアプリの説明（オンボーディングと同じ内容）を再表示

## 2. Non-Functional Requirements

### NFR-01: パフォーマンス / UX
- 起動 → 園追加画面: 2タップ以内
- 園追加（園名入力→保存）: 15秒以内
- 起動 → 既存園のメモ追記開始: 2タップ以内
- メモ保存: 自動保存 or 1タップ
- 親指ゾーンの原則: 主要操作ボタンは画面下部に配置

### NFR-02: データ保存
- localStorage にデータ保存（サーバーに送信しない）
- ストレージキー: `hokatsu-techo`（バージョン共通、`schemaVersion` で構造を管理）
- `schemaVersion` でデータ構造バージョンを管理（v1では1、将来のv2/v3でインクリメント）
- 読み込み時に `schemaVersion` に応じたマイグレーション処理を実行
- 足りないフィールドはデフォルト値で補完

### NFR-03: PWA
- Service Worker + Web App Manifest
- ホーム画面追加対応
- オフライン対応（圏外でも使える）
- 初回起動時に「この端末内に保存されます（機種変更で消えます）」を明示

### NFR-04: アナリティクス
- Google Analytics 4 (GA4) を導入
- Microsoft Clarity を導入
- Cookie同意バナーを実装
- 入力データは分析ツールに送信しない（ページビュー等のみ）

### NFR-05: テスト
- 単体テスト: Vitest
- 結合テスト: Vitest + React Testing Library
- 主要なコンポーネント・フック・サービスのカバレッジを確保

### NFR-06: セキュリティ
- SECURITY拡張ルールを適用（全ルールをブロッキング制約として適用）
- 入力バリデーション（園名の空チェック等）
- localStorageのみ使用（外部通信なし、データはクライアント側のみ）

### NFR-07: アクセシビリティ
- セマンティックHTML
- キーボードナビゲーション対応
- スクリーンリーダー対応（aria属性）

## 3. Technical Specifications

### 技術スタック

| 項目 | 技術 |
|------|------|
| フレームワーク | Next.js |
| スタイリング | Tailwind CSS |
| 言語 | TypeScript |
| テスト | Vitest + React Testing Library |
| デプロイ | Vercel |
| PWA | next-pwa または同等のPWAプラグイン |
| アナリティクス | GA4 + Microsoft Clarity |

### データモデル

```typescript
interface Nursery {
  id: string;              // UUID
  name: string;            // 園名（必須）
  visitDate: string | null; // 見学日（ISO 8601形式）、未定ならnull
  memo: string;            // メモ（空文字列可）
  createdAt: string;       // 作成日時（ISO 8601形式）
  updatedAt: string;       // 更新日時（ISO 8601形式）
}

interface AppState {
  schemaVersion: number;   // データ構造バージョン（v1では1）
  nurseries: Nursery[];    // 園リスト
  hasSeenOnboarding: boolean; // オンボーディング表示済みフラグ
}
```

### 画面構成

| 画面 | パス（想定） | 役割 |
|------|-------------|------|
| 初回オンボーディング | `/` (モーダル/オーバーレイ) | 見学のコツ表示、初回のみ |
| 園一覧（ホーム） | `/` | 登録した園の一覧表示、園追加への導線 |
| 園追加 | `/add` | 園名と見学日を入力して追加 |
| 園詳細 | `/nursery/[id]` | メモの表示・編集、園情報の編集・削除 |

### データ移行
- プロトタイプからのデータ移行機能は含めない（既存ユーザーは保活完了済み）
- v1は新規データのみを対象とする
- 将来のv2/v3へのマイグレーションは `schemaVersion` ベースで対応

## 4. Scope Exclusions (v1に含めないもの)

| 機能 | 理由 | 将来バージョン |
|------|------|---------------|
| 質問テンプレート | なくても「印象を残す」は達成できる | v2候補 |
| はい・いいえ選択式 | なくても「印象を残す」は達成できる | v2候補 |
| 振り返りガイド | なくても「印象を残す」は達成できる | v2候補 |
| 並び替え機能 | v1は追加順固定で十分 | v2候補 |
| 比較機能 | 優先度低い（直感で判断される） | v3以降 |
| パートナー共有 | 有料機能として検討 | 将来 |
| データエクスポート | v1は最小限に | v1.1候補 |
| プロトタイプデータ移行 | 既存ユーザーは保活完了済み、実ユーザーなし | 対応不要 |
